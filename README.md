# 週次制作：模倣から学び、改変して公開する

## 概要
このリポジトリは、「模倣から学び、改変して公開する」というテーマの下で行う週次制作の成果物をまとめたものです。単に「ものを作る」ことだけを目的とせず、既存のサービスやアプリケーションを参考にしながら、自分で理解し、改変し、学びを言語化する力を身につけることを目指しています。

## 制作の基本方針
各週の成果物は、以下のプロセスに沿って進めています。

1. **参考対象の再現**：既存のサービスやツールなどの動きを観察し、まずは動作するアプリケーションとして再現します。
2. **改変の追加**：単なる再現で終わらせず、機能の追加やUIの改善など、必ず1つ以上の改変を加えています。
3. **学びの言語化**：作成後は、参考にした点、自作した点、難しかった点などを振り返りとして整理し、ドキュメントに残しています。

### 生成AIの活用とルール
開発にあたり、生成AIやWeb上のリソースは積極的に活用しています。ただし、生成AIは答えを丸写しするためではなく、実装方針の相談やエラー原因の調査など「理解を助ける道具」として使用しています。
また、公開にあたっては他者の著作権やライセンスを確認し、必ず参考元の出典を明記しています。

## 各プロジェクトのディレクトリ構成
本リポジトリ内の各週のプロジェクト（フォルダ）は、原則として以下の構成で統一しています。

- `README.md` : 各成果物の概要、加えた改変、使った技術、学びなどを簡潔にまとめた説明資料。
- `src/` : 実際のプログラム本体（ソースコード）を格納するディレクトリ。
- `assets/` : READMEなどで使用する画面キャプチャ、図表、デモ動画などの素材。
- `docs/`
  - `reference.md` : 参考にしたサービスやWebページの情報と、そこから何を学んだかの整理。
  - `reflection.md` : 難しかった点、うまくいかなかった点、自分の言葉で説明できるようになったことなど、深い学びと反省の記録。
- `notes/`
  - `weekly-log.md` : 制作途中の日々の作業記録、詰まった点、試行錯誤の過程や生成AIへの質問内容を残すメモ。
